Budget Control Modes  
 

The budget control module features 3 operational modes that specify how the module should behave. Setting of these modes will re-configure your company's buying application. In addition, it has the effect of altering how your company's budgets are monitored. To find out more about each mode, click on it's respective link below.

Introduction
 This module is found in the role of Finance Manager if your company has subscribed to the Budget Control Module (BCM).  When BCM is enabled, company allocated funds are checked and encumbered based on the transaction going through the company.

To effectively utilize this module, the Finance Manager would have to set the BCM mode.  You can find the feature on BCM mode under Budget Control Management.  Select General Settings to access the feature page.  Below you will find brief explanation on the following BCM modes: 

 
     
 

Absolute
When you set BCM mode to Absolute budgeting that means for every PR, PO and Invoice handled by the system, it will check according to a default process flow.  That is, at the point of raising a PR, user (or buyer) would have to assign an account code or sub-account code for the PR.  The system would then check the total value of this PR against the existing budget of the said account.  If the PR value exceeds budget, the system will display a message stating that there is insufficient budget in the account to perform the requisition.

The user (or buyer) would then be prompted to take one of the following corrective actions:

  1. charge to another account code
  2. change quantity
  3. apply for additional budget (when this is selected, the system would email request for additional budget to the finance manager stating account details and reasons for request.)

For PR value that is within the budget of the said account, the system would set aside an amount equivalent to PR value in the budget.

Under the circumstance where PR/PO is rejected by Approving Officer/Vendor, the system would released the reserved amount back to the original budget.

 
     
  Advisory
In advisory control mode, the system would allow user (buyer) to raise a PR and send for approval even if the PR value exceeds budget allocated to the said account. However, being in an advisory mode, the system would alert Finance Manager when budget is exceeded. 		  
     
  None
If this mode is set, the system would allow user (buyer) to raise PR and send for approval even if the PR value exceeds budget allocated to the said account.  No email alert would be sent to Finance Manager.
